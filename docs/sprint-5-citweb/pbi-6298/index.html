Descrição

Criação de relatório consolidado de SLA para acompanhamento dos indicadores de homologação da Fairfax. O relatório deve permitir visualizar prazos, desempenho e cumprimento dos tempos acordados para atendimento, testes e validações.

Critérios de Aceite
  • Sistema deve gerar relatório SLA específico para HML
  • Campos mínimos: data abertura, data início HML, data conclusão, tempo total atendimento, status (dentro/fora SLA), responsável
  • Filtros: período, tipo de demanda, status
  • Atualização automática ou em intervalos definidos
  • Exportação para Excel e PDF
  • Acesso por perfil de usuário
  • Dados consistentes com o sistema
  • Validado em HML antes da liberação
Escopo
Acesso ao relatório, campos obrigatórios, filtros, exportação Excel/PDF, controle de acesso por perfil, consistência de dados
Regras de SLA de outras seguradoras; funcionalidades não listadas nos critérios de aceite
Homologação (HML)
Pré-requisitos
  • Ambiente HML disponível e com dados de teste carregados
  • Usuário com perfil de acesso ao relatório de SLA disponível para testes
  • Usuário sem permissão de SLA disponível para testes de restrição de acesso
  • PBI 6298 publicado em HML

Riscos e Observações
  • CTs CT-SLA-001 e CT-SLA-002 são bloqueantes — sem aprovação, os demais grupos perdem base de validação.
  • CT-SLA-009 (restrição de acesso) deve ser executado para garantir que dados sensíveis de SLA não fiquem expostos.
  • A consistência de dados (CT-SLA-010) depende de registros reais disponíveis no HML; caso não haja massa de dados, pode ser necessário carga prévia.
  • Exportação Excel/PDF deve ser testada em diferentes volumes de dados para verificar performance.
