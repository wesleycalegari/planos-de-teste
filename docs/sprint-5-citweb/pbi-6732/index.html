Descrição

Diversas melhorias do produto RCV foram publicadas na branch main do CITNET e implantadas com sucesso apenas na AXA. Este PBI equaliza o ambiente HDI-Yelum com a baseline da AXA: equalização de código, atualização de banco de dados e scripts pendentes.

Escopo e Critérios de Aceite
  • Ambiente HML atualizado e validado
  • Ambiente Staging atualizado e validado
  • Produção atualizada dentro da janela (seg–qui após 18h)
  • Banco de dados sincronizado com a versão atual (baseline AXA)
  • Nenhum erro crítico após deploy em nenhum ambiente
  • Dependências específicas da seguradora HDI-Yelum validadas
Pré-requisitos Gerais
  • Acesso aos ambientes HML, Staging e Produção HDI-Yelum
  • Scripts de banco de dados homologados e disponíveis
  • Pipeline de deploy configurado para a HDI-Yelum no CITNET
  • Validação em HML e Staging concluída antes do deploy em produção
  • Aprovação formal para janela de produção (seg–qui após 18h)

Riscos e Observações
  • Deploy deve seguir a ordem: HML → Staging → Produção. Não pular etapas.
  • Janela de produção obrigatória: deploy em produção apenas seg–qui após 18h, com aprovação formal.
  • Scripts de banco irreversíveis: validar em HML antes de aplicar em Staging/Produção.
  • Configurações HDI-Yelum: verificar parâmetros específicos da seguradora para não sobrescrever customizações.
  • Rollback: plano de rollback deve estar disponível antes do deploy em produção.
