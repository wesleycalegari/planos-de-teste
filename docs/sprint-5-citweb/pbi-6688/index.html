Descrição

Habilitação do módulo CITNET para a seguradora HDI-Yelum e disponibilização das opções AT&M, CITNET e CONVERSOR no campo "Origem Averbação" do cadastro de apólice RCV (ramo 59). Embarques devem ficar disponíveis no CITNET para HDI-Yelum após a atualização.

Critérios de Aceite
  • Módulo CITNET disponível na tela inicial do CITNET para HDI-Yelum
  • Campo "Origem Averbação" no cadastro de apólice RCV (ramo 59) com as opções: AT&M, CITNET, CONVERSOR
  • Embarques disponíveis no CITNET para HDI-Yelum
Escopo
Habilitação do módulo CITNET para HDI-Yelum; campo Origem Averbação no ramo 59; disponibilidade de embarques no CITNET; regressão em outras seguradoras
Demais ramos que não sejam o ramo 59 (RCV); configurações de Origem Averbação de outras seguradoras que não HDI-Yelum
Homologação (HML)
59 — RCV (Responsabilidade Civil de Veículos)
Pré-requisitos
  • Ambiente HML disponível com a versão HDI-Yelum atualizada (PBI 6688 publicado)
  • Credenciais de usuário HDI-Yelum disponíveis para acesso ao CITNET e ao CITWEB
  • Apólice de ramo 59 (RCV) disponível no CITWEB em ambiente HDI-Yelum para edição
  • Credenciais de outra seguradora (ex: AXA) disponíveis para teste de regressão

Riscos e Observações
  • CTs CT-HDI-001 e CT-HDI-002 são bloqueantes para o módulo CITNET — devem ser executados primeiro.
  • CTs CT-HDI-003, CT-HDI-004 e CT-HDI-005 são independentes entre si, mas todos bloqueantes — a ausência de qualquer opção no dropdown inviabiliza as averbações correspondentes.
  • CT-HDI-006 depende da aprovação de CT-HDI-004 para ser executado com significado.
  • CT-HDI-007 é regressão — deve ser executado para garantir que nenhuma outra seguradora foi impactada pelas alterações de versão HDI-Yelum.
  • Alterações são específicas para o ramo 59 (RCV) e para a seguradora HDI-Yelum — não devem afetar outros ramos ou seguradoras.
