Descrição

Habilitação da seguradora HDI-Yelum no ecossistema CITWEB/CITNET, com dois entregáveis principais:

  1. Ativação do indicativo Ind_Modulo_CITNET para exibir o módulo CITNET na tela inicial do CITWEB para usuários HDI-Yelum.
  2. Inclusão das opções CITNET e CONVERSOR no campo Origem da Averbação do cadastro de apólice do ramo 59 (RCV) no CITWEB.
Contexto técnico (Task 6690): Habilitação via Ind_Modulo_CITNET e atualização do campo Origem da Averbação via tabela tb_ori_avb_cit, criada pelos scripts da Task 6737.
Descoberta durante automação: O campo ddlOrigemAvb é somente leitura (disabled) por design no formulário de apólice — exibe a opção configurada mas não permite alteração via UI após criação. Os CTs 004 e 005 foram adaptados para verificar a presença da opção via JavaScript evaluate.
Escopo

✅ In Scope

  • Exibição do módulo CITNET na tela inicial do CITWEB para usuários HDI-Yelum
  • Campo Origem da Averbação no cadastro de apólice ramo 59 — HDI-Yelum (opções AT&M, CITNET, CONVERSOR)
  • Regressão AXA: CONVERSOR não foi adicionado indevidamente ao ramo 59 AXA
  • Regressão CITNET AXA: login, listagem de apólices e formulário de nova averbação RCV ainda funcionam

❌ Out of Scope

  • Outros ramos da HDI-Yelum (mudança é exclusiva do ramo 59)
  • Execução completa de averbação no CITNET (não alterada neste PBI)
  • Fluxo de faturamento das averbações geradas via CITNET

⚠️ Task 6737: Os scripts create_tb_ori_avb_cit.sql e Ind_Existe_tb_ori_avb_cit.sql precisam ter sido executados no ambiente HML antes dos testes manuais dos Grupos 2 e 3.
  • Ambiente: HML HDI-Yelum (hdi-yelum-hml-faturamento.nsseg.com.br) e AXA (axa-hml-faturamento.nsseg.com.br)
  • CITWEB: usuário FRED / DERF — perfil HDI-Yelum

  • Scripts executados: create_tb_ori_avb_cit.sql e Ind_Existe_tb_ori_avb_cit.sql
