4. Estratégia de Automação

CritérioAutomatizarNão automatizar
Tipo de fluxo Regressão de bugs corrigidos, smoke test, validações de campos obrigatórios, controle de acesso Relatórios PDF, fluxos de fechamento mensal, testes exploratórios
Estabilidade de UI Páginas com IDs estáveis (Web Forms ASP.NET com IDs fixos) Páginas em refatoração ativa ou com selects dinâmicos sem ID estável
Dependência de estado Fluxos que podem ser executados com dados fixos de HML Faturamento (depende de período não fechado) e processos com estado compartilhado
Framework Playwright (Python) — padrão já em uso. Sempre usar page.frames para iframes, não frame_locator() em formulários que navegam internamente.
Armadilha conhecida — iframes CITNET2: O formulário de dados da apólice carrega em iframe. Quando o usuário abre Condições Comerciais, Suspensão ou Subgrupo, o iframe navega internamente sem trocar o atributo DOM src. Por isso:
  • Sempre usar page.frames e filtrar por f.url — nunca por frame_locator("iframe[src*='...']")
  • Após qualquer postback ASP.NET, re-obter o frame antes de interagir
  • Campos aspNetDisabled precisam de removeAttribute('disabled') via JS antes do fill

5. Riscos e Mitigações

RiscoProbabilidadeImpactoMitigação
Dados de HML alterados entre execuções (apólice excluída, vigência vencida) ALTA ALTO Mapear apólices fixas de referência por ramo; validar vigência antes de executar
Período de faturamento já fechado impede testes de FAT ALTA ALTO Testes FAT executados apenas em janela específica do mês; coordenar com time
IDs de campos CC diferem por ramo (descoberto em 06/05/2026) MÉDIA MÉDIO Mapa de IDs por ramo documentado; diagnóstico executado antes de novos ramos
Postback ASP.NET demorado causa falso negativo no teste MÉDIA MÉDIO Aguardar wait_for_load_state('networkidle') após cliques que disparam postback
Regras de negócio diferentes por seguradora (AXA vs KOVR vs outros) ALTA ALTO Este plano cobre apenas AXA HML; outros ambientes requerem plano específico
Linkagem R54/R55 — alterar um ramo afeta o outro silenciosamente MÉDIA ALTO CT-SUS-007 cobre este cenário; incluir verificação do ramo vinculado nos testes de R54
